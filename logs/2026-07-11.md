# ネットワークを STANDARD ティアへ切替 — egress 課金の解消

- **日時**: 2026-07-11
- **プロジェクト**: takano32
- **作業者**: takano32@gmail.com + Claude Code

## 概要

egress 課金(Premium $0.12/GB、7月見込み ¥900/月)への対応として、VM の外部接続を
**STANDARD ネットワークティア**に切り替えた。Standard ティアは**リージョンごとに月 200GB まで無料**
(2023-10-01 開始、2026-07 時点も[料金ページ](https://cloud.google.com/network-tiers/pricing)に現行掲載)のため、
現ペース約 50GiB/月なら **egress 課金は ¥0 になる見込み**。

## 実施内容

### 1. ティア切替(ユーザー作業)

```
gcloud compute instances delete-access-config e2-micro --zone=us-west1-a --access-config-name=external-nat
gcloud compute instances add-access-config e2-micro --zone=us-west1-a --network-tier=STANDARD
```

→ 新 IP **35.212.198.233**(STANDARD・エフェメラル)が付与された。

### 2. IP の整理(Claude 作業、ユーザー承認済み)

- 旧静的 IP `e2-micro` = 8.231.236.96(Premium)を**解放**
  — RESERVED のまま残すと定価 $0.01/時 ≒ ¥1,100/月、かつ STANDARD 構成には再アタッチ不可のため
- 新 IP 35.212.198.233 を**静的に昇格**(STANDARD ティア、IN_USE)。当初 `e2-micro-std` と命名したが、旧名が空いたため**削除→同 IP を `e2-micro` で再予約**して旧名に統一(IP 保持のまま、接続断なし)
  — VM 停止→起動で IP が変わるのを防止。使用中静的 IP の定価はエフェメラルと同額($0.005/時)で、
  5月・6月実績ではクレジットで全額相殺 ¥0

### 3. 追随修正

- VM の `/etc/caddy/Caddyfile`: `8-231-236-96.sslip.io` → **`35-212-198-233.sslip.io`** に書換え、reload
- ローカル `~/.ssh/config`: `Host e2-micro` の HostName を新 IP に更新(旧 IP はコメントで保持)
- known_hosts に新 IP のホスト鍵を登録
