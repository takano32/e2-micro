## 概要

無料枠の e2-micro インスタンスを us-central1 から us-west1 へ移行し、IP を静的化。
あわせてプロジェクト内の不要リソースを棚卸しして削除した。

## 作業内容

### 1. インスタンス停止の確認

- ゲスト OS 内からシャットダウンした `e2-micro-0`(us-central1-a)が GCP 上で `TERMINATED` になったことを確認。

### 2. 無料枠(Always Free)適合チェック

- e2-micro / us-central1 / pd-standard 30GB / 静的 IP なし / スナップショットなし — すべて条件適合を確認。

### 3. us-central1 → us-west1 への移行

| 手順 | 内容 |
|---|---|
| スナップショット作成 | `e2-micro-migrate`(旧ディスク `e2-micro` @ us-central1-a から) |
| ディスク作成 | `e2-micro`(pd-standard 30GB)を us-west1-a に同名で作成 |
| インスタンス作成 | `e2-micro` @ us-west1-a、タグ `http-server,https-server`、`serial-port-enable=true` 引き継ぎ |
| 起動確認 | SSH でホスト名・データ(9.7GB 使用)を確認 |
| 旧リソース削除 | 旧インスタンス `e2-micro-0`・旧ディスク・スナップショットを削除 |

- 新しい外部 IP: **8.231.236.96**(旧: 136.114.244.8)
- 接続コマンド: `gcloud compute ssh --zone us-west1-a e2-micro --project takano32`

### 4. Caddy 設定の更新

- `/etc/caddy/Caddyfile` のドメインを `136-114-244-8.sslip.io` → `8-231-236-96.sslip.io` に変更しリロード。
- `https://8-231-236-96.sslip.io/cxx_chatwork` → 200(証明書取得・検証 OK)、他パスは 404 を確認。
- 注意: ChatWork 側 Webhook URL に旧ドメインを登録している場合は要更新。

### 5. 外部 IPv4 料金の調査と IP の静的化

- 調査結果(公式料金ページ):
  - 使用中の外部 IPv4: **$0.005/時(約 $3.65/月)** — 静的・エフェメラル同額
  - 予約済み未使用の静的 IP: $0.01/時(約 $7.3/月)
  - 無料枠 e2-micro でも外部 IPv4 は課金対象(以前の「免除される」という説明を訂正)
- エフェメラル IP 8.231.236.96 を静的アドレス `e2-micro`(us-west1)として昇格。**料金は据え置き**、IP は再起動でも不変に。
- 注意: 将来インスタンスを削除する際は静的アドレスも削除すること(未使用放置で倍額)。
