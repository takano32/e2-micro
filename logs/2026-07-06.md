## 追記2(同日夜): egress 宛先別カウンタの常設

「egress 約30GB/月の内訳」を継続計測できるよう、VM に nftables カウンタを設置した(カウントのみ・遮断なし)。

- 発生源の特定(接続実測): 送信先は Anthropic API 直行(160.79.104.10)、Cloudflare(Discord ゲートウェイ等)、SSH 応答、メタデータのみ。Caddy 公開分は24時間でジャーナル61行と事実上ゼロ。TX 2.68GB vs RX 94MB という送信偏重から、**ほぼ全量が VM 上の claude + watcher の API 送信**と結論。
- 設置物(コピーをリポジトリ `vm/` に保存):
  - `/etc/nftables.d/egress-acct.nft` — 宛先別カウンタ(anthropic_api / cloudflare_discord / ssh_replies / internal_free / other)。`/etc/nftables.conf` に include 追記、`nftables.service` を enable
  - `/usr/local/bin/egress-report` — MB 表示の読み出しコマンド
  - `/etc/cron.daily/egress-acct` — `/var/log/egress-acct.log` へ日次スナップショット(再起動でカウンタが 0 に戻っても履歴が残る)
- テスト通信で5バケットすべての計上を検証済み。
- 6月分の宛先別実績は事後再構成不可。請求レポートの SKU 別表示(宛先大陸別)が唯一の記録。
